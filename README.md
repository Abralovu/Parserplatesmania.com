# PlatesMania Scraper v2.0

Парсер сайта platesmania.com — собирает фотографии номерных знаков с описанием.

## Что собирает

- Номерной знак
- Фотография (скачивается локально)
- Марка и модель автомобиля
- Регион
- Страна

Результат сохраняется в базу данных SQLite и экспортируется в Excel с фотографиями.

---

## Установка (один раз)

### 1. Установи Python 3.11+
Скачай с https://python.org и установи.

### 2. Скачай проект
```
git clone https://github.com/Abralovu/Parserplatesmania.com
cd Parserplatesmania.com
```

### 3. Создай виртуальное окружение
```
python3 -m venv venv
source venv/bin/activate        # Mac / Linux
venv\Scripts\activate           # Windows
```

### 4. Установи зависимости
```
pip install -r requirements.txt
playwright install chromium
```

### 5. Настрой конфиг
Скопируй `.env.example` и переименуй в `.env`:
```
cp .env.example .env
```

Открой `.env` и укажи путь к своему профилю Chrome:
```
CHROME_PROFILE=~/Library/Application Support/Google/Chrome/Default
```

Как найти свой профиль: Chrome → адресная строка → `chrome://version/` → строка "Путь к профилю".

## Способ 1 — Веб-панель (рекомендуется)

Запусти веб-интерфейс:
```
python main.py web
```

Открой браузер: **http://localhost:8000**

В панели можно:
- Выбрать страну и диапазон ID
- Запустить парсинг в 1, 3 или 6 потоков
- Следить за прогрессом в реальном времени
- Смотреть и фильтровать собранные данные
- Скачать Excel или CSV с фильтрами
- Просмотреть историю сессий
- Полностью сбросить все данные

### На VPS (без GUI)
В `.env` измени:
```
HEADLESS=true
```
Браузер будет работать в фоне без окна.

## Способ 2 — Командная строка

### Запуск парсинга
```
python main.py scrape --country ru --start 31000000 --end 31100000
```

Параметры:
- `--country` — код страны: `ru`, `de`, `us`, `kz`, `ua`, `by`, `uz`, `pl`, `fr` и др.
- `--start` — начальный ID
- `--end` — конечный ID
- `--workers N` — количество потоков (по умолчанию 1)
- `--fresh` — начать заново, игнорируя прогресс
- `--auto` — автоматически определить диапазон с сайта

### Многопоточный запуск
```
python main.py scrape --country ru --start 31000000 --end 31100000 --workers 3
```

### Экспорт в Excel с фотографиями
```
python main.py excel --country ru
```
Файл сохранится в `data/plates_ru.xlsx`

### Экспорт в CSV
```
python main.py export --country ru
```

### Статус — сколько собрано
```
python main.py status --countries ru,de,us
```

### Сброс прогресса
```
python main.py reset --country ru
```

## Где хранятся данные
```
data/
├── plates.db          ← база данных со всеми записями
├── checkpoint.json    ← прогресс парсинга (не удалять!)
├── history.json       ← история сессий
├── plates_ru.xlsx     ← Excel выгрузка
├── plates_ru.csv      ← CSV выгрузка
├── profiles/          ← Chrome профили (создаются автоматически)
└── photos/
    └── ru/
        ├── 31000001.jpg
        ├── 31000002.jpg
        └── ...
```

**Важно:** папку `data/` не удалять — там все собранные данные и прогресс.

## Частые вопросы

**Парсер остановился — что делать?**
Запусти снова ту же команду без `--fresh`. Продолжит с места остановки автоматически.

**Открывается окно браузера — это нормально?**
Да. Сайт использует защиту от ботов, парсер работает через реальный Chrome. Окно можно свернуть, не закрывать. На VPS установи `HEADLESS=true` в `.env`.

**Почему не все ID дают результат?**
Не все ID существуют на сайте — это нормально. Парсер автоматически пропускает пустые страницы.

**Где взять диапазон ID для России?**
Последние загруженные номера видны на главной странице сайта. Актуальный максимум для России около 31 400 000. Рекомендуется парсить диапазонами по 100 000.

**Как парсить другую страну?**
```
python main.py scrape --country de --start 1 --end 500000
```

## Скорость парсинга

| Режим | Скорость |
|-------|----------|
| 1 поток (Тихо) | ~600 записей/час |
| 3 потока (Средне) | ~1800 записей/час |
| 6 потоков (Быстро) | ~3600 записей/час |

Скорость зависит от интернета и задержек сайта. Старые ID (до 1 000 000) заполнены редко — парсер будет часто получать пустые страницы, это нормально.
